# Mem0: Building Production-Ready AI Agents with Scalable Long-Term Memory

- arXiv: 2504.19413 (April 2025)
- URL: https://arxiv.org/abs/2504.19413
- PDF: https://arxiv.org/pdf/2504.19413
- Type: academic-paper (industry, Mem0.ai)

## One-line claim
A memory-centric architecture that extracts, consolidates, and retrieves salient facts
from ongoing conversations, replacing full-context replay — cutting token cost >90% and
p95 latency 91% while improving answer accuracy 26% over OpenAI's memory.

## Architecture
- Two-stage pipeline:
  1. **Extraction** — an LLM extracts key facts/salient information from the latest
     conversation turns.
  2. **Update** — an LLM compares extracted facts against existing memory and issues one
     of `ADD / UPDATE / DELETE / NOOP` operations to keep the memory store consistent
     (avoids duplicate or contradictory memories).
- **Mem0g** (graph variant): stores memories as a graph capturing relational structure
  among conversational entities; ~2% higher overall score than base Mem0.
- Retrieval: at query time, semantically relevant memories are fetched and injected into
  the prompt instead of the full history.

## Key numbers (LOCOMO benchmark)
- **+26%** relative improvement in LLM-as-a-Judge accuracy over OpenAI memory.
- **−91%** p95 latency vs full-context approach.
- **>90%** token-cost savings vs feeding the entire conversation history.
- **Mem0g**: ~**+2%** overall score over base Mem0.
- Evaluated against six baseline families: memory-augmented, RAG variants, full-context,
  open-source, proprietary, and dedicated memory platforms; task types: single-hop,
  multi-hop, temporal, open-domain.

## Relevance to on-device agent memory
Full-context replay is the "original" baseline whose token cost and latency grow with
history length — fatal on bandwidth-bound mobile decode. Mem0 keeps the live context (and
therefore the KV cache) small by externalizing memory and retrieving only what is needed.
